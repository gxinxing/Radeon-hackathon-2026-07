## 1. Target Application Definition

We train a humanoid robot soccer policy entirely in simulation, accelerated by
AMD Radeon GPUs and the ROCm software stack. The task design is anchored on the
real **Booster T1 / RoBoLeague 3v3** humanoid soccer league: a single robot (and,
later, a 3v3 team) must chase the ball, dribble, shoot toward the goal, recover
from falls, and coordinate attacker/defender roles. The goal is a reproducible,
GPU-accelerated embodied-AI benchmark that mirrors a real robot-soccer competition.

## 2. Overall System Architecture and Solution Design

- **Simulation**: Genesis with the GPU backend (ROCm), headless, batched parallel
  environments (`num_envs`) for throughput.
- **Policy**: shared-trunk MLP actor-critic; Gaussian continuous actions mapped to
  joint targets.
- **RL algorithm**: PPO, trained as a curriculum — `balance → chase → dribble →
  shoot → coop` — to keep early training stable before adding ball dynamics and
  multi-agent interaction.
- **Reward**: Booster-derived curriculum (see `rewards/reward.py`) turning observed
  soccer behaviors (upright, chase, ball control, shoot, recovery) into RL terms.
- **Multi-agent**: a single role-conditioned shared policy (attacker / defender flag
  in observation) instead of separate networks — cheap and demo-friendly.
- **Optional VLA layer**: a natural-language command → action layer
  (SmolVLA / OpenVLA) running on Radeon, for the "innovation" criterion.
- Architecture diagram: `docs/ARCHITECTURE.md`.

## 3. Datasets for Training and/or Evaluation

Training is online reinforcement learning inside the simulator; no external dataset
is required. If an imitation-learning variant is added, demonstration trajectories
are generated by the simulator and exported to `LeRobotDataset` format.

## 4. Utilization of AMD Radeon GPUs (training / inference)

- All training and inference run on a **single AMD Radeon GPU** via the ROCm
  software stack (PyTorch ROCm wheel). Under ROCm, `torch.cuda` maps to the AMD GPU.
- GPU-parallel simulation (`num_envs` batched) maximizes throughput.
- `scripts/benchmark_gpu.py` reports device name, ROCm version, matmul TFLOPS,
  environment steps/sec, and peak memory; cross-checked with `rocm-smi`.
- This directly satisfies the Track 3 "AMD Radeon GPU + ROCm adoption" criterion.

## 5. Innovations and Key Technical Contributions

- A **Booster-match-inspired reward curriculum** that maps real soccer behaviors
  observed in RoBoLeague into RL reward terms.
- **Role-conditioned multi-agent coordination** (attacker / defender) using one
  shared policy with a role flag — no separate networks.
- (Optional) A **lightweight VLA instruction layer** that runs on Radeon with
  measurable latency, closing the loop from language command to action.
